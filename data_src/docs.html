-------------

Carts
==========

http://localhost/data_src/api/cart/read.php?APIKEY=secretKEYhere
http://localhost/data_src/api/cart/read.php?APIKEY=secretKEYhere&id=cart_id
http://localhost/data_src/api/cart/create.php
http://localhost/data_src/api/cart/update.php

Under the data_src folder we are using a REST API created using Core PHP. All CRUD operations on the database will be done using this code.

This is API will use an API key for validation that will change in production. If you confront any error, please contact me at GitHub

While you're trying to Create, Update, or Delete through API, make sure your provided data are in JSON format because header only accepts Content-Type: application/json in these requests.

Products

These are items available at the food pantry. The quantiy available and the category are fields as well as an image path.

Get Products

To get products you need to make a GET call to the following url :
http://localhost/data_src/api/product/read.php?APIKEY=secretKEYhere 


Get Product Result Format Example :

[
    {
        "productID": "201",
        "productName": "Sliced Carrots",
        "catID": "2",
        "categoryName": "Canned Goods",
        "quantity": "55",
        "img": "/path/to/image/imgFileName.jpg",
    } ...
]

This will provide all the products. So, you don't have to pass any QUERY PARAMETERS

Make Sure you add the API key

You need to add the API KEY to the all the web requests and urls as follows:
http://localhost/data_src/api/product/read.php?APIKEY=secretKEYhere


Get a Single Product

To get a specific product you need to make a GET call to the following url :
http://localhost/data_src/api/product/read.php?APIKEY=secretKEYhere&id=product_id 


The URL should look like:


http://localhost/data_src/api/product/read.php?APIKEY=secretKEYhere&id=201

            
Result if any product exists:

[
  {
    "productID": "201",
    "productName": "Sliced Carrots",
    "catID": "2",
    "categoryName": "Canned Goods",
    "quantity": "55",
    "img": "/path/to/image/imgFileName.jpg",
  }
]

Just pass the product id to get the specific product from API

Get Products for a specific category id

To get products you need for a specific category make a GET call to the following url :
http://localhost/data_src/api/product/read.php?APIKEY=secretKEYhere&catID=category_id 


Get Product Result Format Example :

[
    {
        "productID": "201",
        "productName": "Sliced Carrots",
        "catID": "2",
        "categoryName": "Canned Goods",
        "quantity": "55",
        "img": "/path/to/image/imgFileName.jpg",

Create a Product

To create a new product you need to make a POST request to the following url :
http://localhost/data_src/api/product/create.php 



Create Product JSON format Example :

{

    "APIKEY": "string",
    "catID": int,
    "productName": "string",
    "quantity": int,
    "img": "string",
}

On Success, Response Message Will Be:

{
    "message": "Product Created Successfully"
}

QUERY PARAMETERS

Field Type Description
APIKEY String Secret API Key provided
catID Integer Product Category
productName String Your product title
quantity Integer number of products in pantry
img String img file path and name

Update a Product

To update a new product you need to make a PUT request to the following url :
http://localhost/data_src/api/product/update.php 



Update Product JSON format Example :

{
    "APIKEY": secret API Key,
    "productID": 0,
    "catID": int,
    productName": "string",
    "quantity": int,
    "img": "string",
}

On Success, The Response Will Be:

{
    "message": "Product Updated Successfully"
}

QUERY PARAMETERS

Field Type Description
APIKEY String Secret API Key
productID Integer Existing Product Id
catID Integer Product Category ID
productName String Your product title
quantity Integer number of products in pantry
img String img file path and name

Carts

All the processes for Carts are same as Products!

Get Items From Every Cart

To get cart items you need to make a GET request to the following url :
http://localhost/pantry2/data_src/api/cart/read.php? APIKEY=secretKEYhere 



            
Get Cart Item Result Format Example for Every Cart:
[
    {
        "basketItemID": "1",
        "productID": "201",
        "basketID": "16",
        "quantity": "55",
    } ...
]

This will provide all cart items. So, you don't have to pass any QUERY PARAMETERS

Get Items From A Specific Cart

To get cart items you need to make a GET request to the following url :
http://localhost/pantry2/data_src/api/cart/read.php? APIKEY=secretKEYhere&id=basketID 


The URL should look like:

            
            http://localhost/pantry2/data_src/api/cart/read.php?APIKEY=secretKEYhere&id=1
            
                        
            Result for each item from the given cart:
            
            [
              {
                "basketItemID": "1",
                "productID": "201",
                "basketID": "16",
                "quantity": "55",
              }
            ]

Just pass the basket id to get the specific cart from API

Create New Cart

To create a cart for a new user you need to make a POST request to the following url:
http://localhost/pantry2/data_src/api/cart/create.php 



Create New Cart JSON format Example :
{
  "APIKEY": nsW3a567
  "userID": 1
}

Success Response Will Be
{
    "message": "New Basket Created, basketID"
}
If the user already has a cart
{
  "message": "Using Current Basket, basketID"
}

QUERY PARAMETERS

Field Type Description
APIKEY String Secret API Key
userID Integer The user's ID the new cart is assigned to

Checkout Cart

To checkout a cart you need to make a POST request to the following url :
http://localhost/data_src/api/cart/checkout.php 



Checkout Cart JSON format Example :
{
  "APIKEY": nsW3a567
  "id": 0
}

Success Response: 200 Status Code

QUERY PARAMETERS

Field Type Description
APIKEY String Secret API Key
id Integer The Id of the cart to be checked out

Cart Items

All the processes for Cart Items are same as Products!

Delete a Whole Cart

To delete a whole cart you need to make a DELETE request to the following url :
http://localhost/pantry2/data_src/api/cart_item/delete.php 



Remove Cart Item JSON format Example :
{
  "APIKEY": nsW3a567
  "basketID": 16
}

Success Response Will Be
{
    "message": "Cart Successfully Deleted!"
}
If the cart doesn't exist in the database, the response message will be
{
  "message": "Cart Does Not Exist!"
}

QUERY PARAMETERS

Field Type Description
APIKEY String Secret API Key
basketID Integer cart that is being deleted

Remove a Cart Item

To remove an item from a cart you need to make a DELETE request to the following url :
http://localhost/pantry2/data_src/api/cart_item/delete.php 



Remove Cart Item JSON format Example :
{
  "APIKEY": nsW3a567
  "productID": 0
  "basketID": 16
}

Success Response Will Be
{
    "message": "Cart Item Removed Successfully!"
}
If the cart item doesn't exist in the database, the response message will be
{
  "message": "Cart Item Does Not Exist!"
}

QUERY PARAMETERS

Field Type Description
APIKEY String Secret API Key
productID Integer product id of the item being removed
basketID Integer cart that the item is being removed from

Users

All the processes for Users are same as Products!

Create a User

To create a new user you need to make a POST request to the following url :
http://localhost/data_src/api/user/create.php 



Create User JSON format Example :

{

    "APIKEY": "string",
    "username": "string",
}

On Success, Response Message Will Be:

{
    "userID": 1
}

If the studentID is already taken, the response will be:

{
  "status": error,
  "message": This student ID is already registered
}

QUERY PARAMETERS

Field Type Description
APIKEY String Secret API Key provided
username String The username of the user
